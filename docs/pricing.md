# 价格说明

本平台支持多种大模型 API 接入，主要采用两种计费方式：

1. **文本类模型按 Tokens 收费**
2. **图像类模型按调用次数收费**

> API 平台地址：<https://jeniya.cn>

> API 模型价格：<https://jeniya.cn/pricing>


## 计费原则

不同模型和分组的成本存在差异，因此实际价格也会不同。一般来说：

- 能力更强、上下文更长的模型，价格通常更高
- 轻量模型更适合高频调用和成本控制
- 图像模型通常按调用次数收费，而不是按 Tokens 收费
- 不同分组由于上游渠道和可用性不同，费率也会不同

## 文本模型计费方式

文本模型通常按输入 Tokens 和输出 Tokens 进行计费，常见适用模型包括：

- GPT 系列
- Claude 系列
- Gemini 文本模型
- DeepSeek、Qwen 等国产模型

**适合场景：**

- 聊天问答
- 文本生成
- 文案写作
- 代码生成
- 知识库问答
- 工作流自动化

## 图像模型计费方式

图像生成类模型通常按调用次数收费，例如：

- gpt-image-2
- Gemini 部分图像模型

**适合场景：**

- AI 生图
- 封面图生成
- 宣传图生成
- 视觉内容辅助创作

## 分组费率说明

不同分组对应不同上游渠道、成本和适用场景。

| 分组 | 类型说明 | 费率 | 支持模型 |
| --- | --- | --- | --- |
| default 默认 | 混合 ChatGPT(AZ渠道) + Claude(逆向渠道) + MJ(快速模型) + 国产模型(DeepSeek/Qwen) | 官方费率 ×1 | OpenAI、Claude、国产模型 |
| 企业级高可用大模型 | 国产模型优先，强调稳定与可用性 | 官方费率 ×1 | 国产模型 |

| 纯AZ | 只有 ChatGPT(AZ渠道) + 国产模型 | 以平台实际说明为准 | ChatGPT、国产模型 |

## 分组选择建议

### 如果你是普通开发者或中小项目
优先考虑：

- default 默认
- 优质gemini
- 国产模型相关分组

### 如果你更重视稳定性和企业可用性
优先考虑：

- 企业级高可用大模型

- 优质grok

## 常见计费问题

### Token 是怎么计算的？
文本模型通常按输入和输出的 Tokens 分别计费。你发送给模型的内容会消耗输入 Tokens，模型返回的结果会消耗输出 Tokens。

### 为什么不同模型价格差别很大？
不同模型在能力、速度、上下文长度、上游资源和供应渠道上存在明显差异，因此成本不同。

### 为什么不同分组费率不一样？
不同分组使用的上游渠道、账号资源、可用性和运维成本不同，所以费率会不同。

### 图像模型为什么按次数收费？
图像模型的成本通常按单次生成计算，因此更适合按调用次数计费。
